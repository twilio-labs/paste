## Guidelines

### About Badge

A badge is a small chunk of information that describes an object in a visually highlighted way. You can think of badges as a 1-3 word description of a more complex object that is highlighted to draw visual focus. Badges should be used to describe static attributes.

## Composition notes

### children

- string, icon, Text, Box
- will resize the icons
- cannot be empty

### display should be inline

- should not create multi-line badges
- should set the badge to be inline if passing multiple children to the badge

<DoDont>
  <Do title="Do" body="Use 1-3 words as badge text.">
    <Badge variant="new" as="span">
      <NewIcon title="New badge component" size="sizeIcon10" />
      New Component
    </Badge>
  </Do>
  <Dont title="Don't" body="Don't use lengthy copy inside a badge.">
    <Badge variant="error" as="span">
      This is some very long text inside a badge that creates a very large badge.
    </Badge>
  </Dont>
</DoDont>

<DoDont>
  <Do title="Do" body="Set the badge children to be inline.">
    <Badge variant="new" as="span">
      Inline badge
    </Badge>
  </Do>
  <Dont title="Don't" body="Don't use multiline badge text.">
    <Badge variant="new" as="span">
      This
      <Box as="br" /> is <Box as="br" />
      not ideal
    </Badge>
  </Dont>
</DoDont>

### API

#### Installation

```bash
yarn add @twilio-paste/badge - or - yarn add @twilio-paste/core
```

#### Usage

```jsx
import {Badge} from '@twilio-paste/badge';

const BadgeExample = () => (
  <Badge variant="default" as="span">
    Default Badge
  </Badge>
);
```
