## üì£ What's new in Paste

### Chat Log animations

<Box as={Image} maxWidth="size60" src={ChatLog} alt="" style={{objectFit: 'contain'}} />

The final piece of [the Chat Log package](/components/chat-log) is complete‚Äî animation! There is a new useChatLogger hook that manages a list of messages, bookends, and events and a new ChatLogger component that animates the list into a Chat Log.

<Paragraph>
  <Anchor href="/components/chat-log#usechatlogger-hook">Check out the useChatLogger docs</Anchor>
</Paragraph>

### In Page Navigation

<Box as={Image} maxWidth="size60" src={PageNav} alt="" style={{objectFit: 'contain'}} />

The In Page Navigation component is a collection of links that navigate users to related pages. Although stylistically similar to Tabs, this component‚Äôs structure allows for straightforward integration with routers like React Router.

<Paragraph>
  <Anchor href="/components/in-page-navigation">Check out the In Page Navigation docs</Anchor>
</Paragraph>

### Chat Composer

<Box as={Image} maxWidth="size60" src={ChatComposer} alt="" style={{objectFit: 'contain'}} />

A highly anticipated component, the Chat Composer is a rich text input that users can type chat messages into. It is built with [the Lexical text editor](https://lexical.dev/) and can be extended to include the features you need through Lexical‚Äôs plugins.

<Paragraph>
  <Anchor href="/components/chat-composer">Check out the Chat Composer docs</Anchor>
</Paragraph>

### Display Heading

<ResponsiveImage src={DisplayHeading} />

Many marketing pages use large text to call attention to information or express their brand. Display Heading is a new component that is meant to be used for large text that does not provide any hierarchical structure to the page.

<Paragraph>
  <Anchor href="/components/display-heading">Check out the Display Heading docs</Anchor>
</Paragraph>

### Additional updates

#### New

- Paste now uses Twilio Sans Mono for monospace text! Check it out in the [Inline Code](/components/inline-code) and [Code Block](/components/code-block) components.
- [Localization documentation](/foundations/localization) has been updated with info on how to internationalize Paste components.
- [Word list](/foundations/content/word-list) provides guidance on how, when, and whether to use specific words, as well as guidance on capitalization for product-specific terms. Reach out to **Megan Allison** and **Aren Goetcherian** to learn more.

Justin registered Paste React components to Builder.io which allows Justin and his team to rapidly create rich Paste-based experiences in the Flex UI using a drag and drop editor. By exposing paste component properties like variants in the Builder.io editor, Justin can choose between different styles of buttons, alerts and any other Paste component that they have wired up. As you can see by the video demo's the authoring is easy and intuitive and the customer experiences and functionality it creates are excellent and fully functional. Another benefit to his team is that they can begin to build complex components using many different Paste components, and save them as templates for easy re-use in customer demos.
