## ðŸ“£ Everything in the [Navigation UI Kit](/patterns/navigation)

The Navigation UI kit structures common elements found in Twilio products to provide customers a cohesive, expected way to find what they're looking for. Applying this pattern across all our experiences also makes the Twilio product suite feel and behave like one family of products.

The Navigation UI Kit is composed of a collection of ingredients that can be combined in different ways to create navigation structure for a variety of applications.

<ResponsiveImage src={NavUIKitImage} alt="Preview of the Navigation UI Kit components" />

### [Topbar](/components/topbar)

<ResponsiveImage src={TopbarIngredientsImage} alt="Topbar Ingredients" />

<ResponsiveImage src={TopbarImage} alt="Preview of the Topbar component" />

Our Topbar is fixed at the top of your app window for easy access to key application controls. It conveniently categorizes controls into two sections: User and Account. To keep things organized, we recommend separating these controls; user controls on the right and account controls on the left.

Account control examples include:

- [Account Switcher](/components/account-switcher)
- [Online Status controls](/components/status-menu)
- Account type indicators, such as subaccount [Badges](/components/badge)
- Possible upgrade actions

Here are some user control possibilities:

- Search function using [Input](/components/input) or [Combobox](/components/combobox)
- Support [Menu](/components/menu)
- [Product Switcher](/components/product-switcher)
- [User Dialog](/components/user-dialog)

### [Sidebar](/components/sidebar)

<Box display="flex">
  <ResponsiveImage src={SidebarIngredientsImage} alt="Preview of the Navigation UI Kit components" />
  <ResponsiveImage src={SidebarImage} alt="Preview of the Navigation UI Kit components" />
</Box>

The Sidebar forms the heart of primary navigation within any application. It caters to both flat and hierarchical structures, thereby meeting the diverse needs of different Twilio applications. Its consistent design ensures a familiar way to navigate back to an applications home dashboard across all Twilio products. With two collapsible modesâ€”"push" and "overlay"â€”it intends to cater to the needs of distinct, and different user personas based on their screen real estate needs.

### [Account Switcher](/components/account-switcher)

<Box maxWidth="400px">
  <ResponsiveImage src={AccountSwitcherImage} alt="Preview of the Navigation UI Kit components" />
</Box>

An essential part of any Twilio application is the ability to switch between Accounts, Sub-accounts, Workspaces or Projects. The Account Switcher offers a uniform method of changing accounts across all applications, fostering familiarity even as users switch between distinct products.

### [Product Switcher](/components/product-switcher)

<Box maxWidth="400px">
  <ResponsiveImage src={ProductSwitcherImage} alt="Preview of the Navigation UI Kit components" />
</Box>

Transitioning between different products or applications is another key feature required by our customers as they weave their engagement solutions together with Twilio's multitude of capabilities and APIs. The Product Switcher ensures a consistent and intuitive means for navigating through purchased Twilio services and can be located conveniently in the [Topbar](/components/topbar).

### [User Dialog](/components/user-dialog)

<Box maxWidth="400px">
  <ResponsiveImage src={UserDialogImage} alt="Preview of the Navigation UI Kit components" />
</Box>

Providing a common way for a user to control or edit their account, settings or preferences is the job of the User Dialog. Designed to be housed in the [Topbar](/components/topbar), the User Dialog provides you with a way to display user data such as an Avatar, email address and name, along with actions that user might wish to take. Actions might include editing user settings, visiting documentation, switching between light and dark modes, or languages and logging out.

### [Status Badge](/components/status-badge)

<ResponsiveImage src={StatusBadgeImage} alt="Preview of the Navigation UI Kit components" />

The [Status Badge](/components/status-badge) further evolves from our [Status Pattern](/pattern/status), allowing you to communicate status updates for any process or connection seamlessly. It provides an implementation of the Status Pattern using an appropriate combination of status level indicators, icons and colors.

### [Status Menu](/components/status-menu)

<Box maxWidth="400px">
  <ResponsiveImage src={StatusMenuImage} alt="Preview of the Navigation UI Kit components" />
</Box>

For processes that require frequent changes in status, we've married together the [Menu](/components/menu) with the [Status Badge](/components/status-badge). The resulting Status Menu allows quick understanding and modification of ongoing statuses.
